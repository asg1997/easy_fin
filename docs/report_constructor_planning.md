# Конструктор расчётов (шаблоны отчётов)

## Цель

Дать пользователю возможность собрать **именованный шаблон отчёта** — отдельно для **прихода** или для **расхода** — и показывать его на вкладке «Отчеты».

Конфиг шаблона хранится **локально** (SharedPreferences), в GitHub sync не попадает. Суммы считаются на лету из SQLite.

---

## Принятые решения

| Вопрос | Решение |
|--------|---------|
| Смешанный приход+расход в одном шаблоне | **Нет.** Шаблон либо приход, либо расход |
| Хранение | SharedPreferences |
| Синхронизация шаблонов | Нет |
| Тип на странице | Переключатель **Приход \| Расход** |
| Название шаблона | **Обязательно** |
| Счёт | **Касса** или **Банк**; поле видно **только если выбрана одна база** |
| База = «Все базы» | Поле счёта **скрыто** |
| Несколько арендаторов / категорий в отчёте | **Отдельная строка** на каждого + строка **Итого** |
| Где создавать шаблон | Только в **Настройках** |
| Период (месяц) | Только при **просмотре** на «Отчетах», в шаблоне не хранится |
| «Все арендаторы» | **Да**, нужна такая опция/галочка |
| Пустой список арендаторов или категорий | **Нельзя сохранить** |
| Приход: взаиморасчёты и прочее | **Только один вид** на шаблон (либо арендаторы, либо категории прочего) |

---

## Экран конструктора

Вход: **Настройки → Шаблоны отчётов** → создать / редактировать.

### Общие поля

| Поле | Поведение |
|------|-----------|
| **Название** | Обязательно |
| **Тип** | Приход / Расход |
| **База** | По умолчанию все базы; можно одну |
| **Счёт** | Только при одной базе: Касса / Банк |

### Приход

1. Вид источника: **Взаиморасчёты** или **Прочее** — строго один вид на шаблон.
2. Дальше:
   - Взаиморасчёты → список арендаторов + опция **«Все арендаторы»**; без выбора сохранить нельзя.
   - Прочее → мультивыбор категорий прочих приходов; пусто сохранить нельзя.

### Расход

- Мультивыбор категорий расходов; пусто сохранить нельзя.

### Схема

```
┌─────────────────────────────────────────┐
│  Название: [________________] *         │
│                                         │
│  Тип:  ( Приход )  ( Расход )           │
│                                         │
│  База:  [ Все базы          ▾ ]         │
│  Счёт:  [ Касса / Банк      ▾ ]         │
│         ↑ только если выбрана одна база │
│                                         │
│  ── Приход ──────────────────────────   │
│  Вид:   ( Взаиморасчёты ) ( Прочее )    │
│                                         │
│  Взаиморасчёты:                         │
│    [✓] Все арендаторы                   │
│    или список арендаторов (мультивыбор) │
│                                         │
│  Прочее:                                │
│    Категории прихода (мультивыбор)      │
│                                         │
│  ── Расход ──────────────────────────   │
│  Категории расхода (мультивыбор)        │
│                                         │
│              [ Сохранить ]              │
└─────────────────────────────────────────┘
```

## Как выглядит таблица на «Отчетах»

Пример: шаблон прихода, взаиморасчёты, выбраны Иванов и Петров, период — июнь.

| | Сумма |
|--|--:|
| Иванов | 50 000 |
| Петров | 30 000 |
| **Итого** | **80 000** |

То же для категорий: по строке на каждую выбранную категорию + Итого.

Период (месяц) и, при необходимости, уточнение базы для просмотра — на вкладке Отчеты, не в шаблоне.

---

## Как шаблон живёт дальше

1. Создание/редактирование только из Настроек.
2. Сохранение → JSON в SharedPreferences.
3. На «Отчетах» — таблица с названием шаблона; суммы из документов + банковских операций.

## Модель конфига (черновик)

```json
{
  "id": "uuid",
  "name": "Приходы арендаторов",
  "kind": "income",
  "baseId": null,
  "accountType": null,
  "income": {
    "sourceKind": "renters",
    "allRenters": false,
    "renterIds": [1, 2]
  }
}
```

```json
{
  "id": "uuid",
  "name": "Все взаиморасчёты",
  "kind": "income",
  "baseId": 5,
  "accountType": "bank",
  "income": {
    "sourceKind": "renters",
    "allRenters": true,
    "renterIds": []
  }
}
```

```json
{
  "id": "uuid",
  "name": "Коммуналка",
  "kind": "expense",
  "baseId": 5,
  "accountType": "cash",
  "expense": {
    "expenseCategoryIds": [1, 3]
  }
}
```

- `baseId: null` — все базы → `accountType` тоже `null`.
- `accountType`: `"cash"` | `"bank"` | `null`.
- При `allRenters: true` список `renterIds` игнорируется; сохранение допустимо.
- При `allRenters: false` нужен непустой `renterIds`.
